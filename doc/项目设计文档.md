# 荔枝小秘书（Lychee Assistant）— 项目设计文档

> 本文档基于 [README.md](../README.md) 中的需求，给出系统的整体架构、功能模块划分、数据库设计与接口设计，作为后续编码实现的依据。

---

## 1. 项目概述

| 项 | 说明 |
| --- | --- |
| 项目名称 | lychee（荔枝小秘书 / Lychee Assistant） |
| 定位 | 辅助 AI 编程的工具网站 |
| 技术栈 | Gin + GORM + SQLite；前端使用 Go 标准 `html/template` 及简易 Jet 模板 |
| 认证方案 | 邮箱 + 密码，JWT 双 Token（Access + Refresh）无感刷新 |
| 多语言 | 中文 / 英文，默认跟随系统语言，可手动切换 |

## 2. 系统架构

### 2.1 分层架构

采用经典的四层结构，职责清晰、便于维护：

```
HTTP 请求
   │
   ▼
[ Router        ]  路由注册、分组
   │
   ▼
[ Middleware    ]  JWT 鉴权 / 限流 / i18n / 操作日志 / 跨域
   │
   ▼
[ Controller    ]  参数校验、请求响应组装
   │
   ▼
[ Service       ]  核心业务逻辑（一个用例对应一个方法）
   │
   ▼
[ Repository    ]  数据访问层（封装 GORM 查询，统一 user_id 隔离）
   │
   ▼
[ Model / DB    ]  GORM 模型 + SQLite
```

### 2.2 目录结构

```
lychee/
├── cmd/server/main.go          # 程序入口
├── internal/
│   ├── config/                 # 配置加载（viper 读取 config.yaml + 环境变量）
│   ├── router/                 # 路由注册
│   ├── controller/             # HTTP 控制器（薄层）
│   ├── service/                # 业务逻辑
│   ├── repository/             # 数据访问层
│   ├── model/                  # GORM 模型定义
│   ├── middleware/             # JWT、限流、i18n、操作日志等中间件
│   └── pkg/                    # 内部工具：响应封装、错误码、JWT、邮件、验证码
├── migrations/                 # 数据库迁移脚本 / AutoMigrate 注册
├── web/
│   ├── templates/              # html/template & Jet 模板
│   └── static/                 # css / js / img 静态资源
├── doc/                        # 项目文档
├── config.yaml                 # 配置文件
├── go.mod
└── README.md
```

## 3. 功能模块设计

### 3.1 多语言模块（i18n）

- **职责**：提供中/英文界面文案，支持运行时切换。
- **实现要点**：
  - 文案以键值对维护，按语言拆分为 `zh.yaml` / `en.yaml`（或内嵌 JSON）。
  - 服务端通过中间件解析语言优先级：**显式切换（Cookie/请求头）> 系统语言（`Accept-Language`）> 默认中文**。
  - 模板渲染时注入翻译函数 `t(key)`，前端切换语言时写入 Cookie 并刷新。
  - 预留扩展接口，后续新增语种只需增加文案文件。

### 3.2 认证模块（Auth）

- **职责**：注册、登录、登出、忘记密码、修改密码、Token 刷新。
- **核心逻辑**：
  - **注册**：邮箱 + 密码 + 邮箱验证码。先调用「发送验证码」接口，校验通过后创建用户。
  - **登录**：校验邮箱 + 密码（bcrypt），签发 Access Token（短有效期，如 15 分钟）与 Refresh Token（长有效期，如 7 天）。
  - **无感刷新**：前端在 Access Token 过期前/过期时，用 Refresh Token 调用刷新接口换取新的 Access Token；Refresh Token 采用「一次性、轮换」策略（rotate）。
  - **数据隔离**：所有业务接口从 JWT 中解析 `user_id`，Repository 层所有查询强制带 `WHERE user_id = ?`。
  - **忘记密码 / 修改密码**：均通过邮箱验证码校验后重置密码。

### 3.3 常用语模块（Phrase）

- **职责**：用户私有常用语（用于在任务编辑时快速插入）的增删改查。
- **要点**：数据按 `user_id` 隔离；列表支持按更新时间倒序。

### 3.4 任务管理模块（Task）

- **职责**：管理任务提示词，左右两栏交互。
- **核心设计**：
  - **分组（task_groups）**：支持自引用 `parent_id` 实现任意层级嵌套；删除分组需级联处理（删除子分组及其下任务，或迁移任务，二选一，默认级联软删除）。
  - **任务（tasks）**：属于某个分组，包含内容、优先级（P0–P5）、状态（编辑中 / 待执行 / 已完成）、截止日期、创建/修改时间。
  - **排序**：支持按创建时间、修改时间、截止日期、优先级排序，配合升降序参数。
  - **编辑弹框**：可修改内容、优先级、状态；可选常用语插入到内容输入框光标位置（前端实现插入，后端只负责常用语列表与任务更新）。
  - **复制**：前端调用剪贴板 API 复制任务内容（无需后端接口，亦可提供 `/copy` 接口返回内容）。

### 3.5 打卡健康模块（Check-in）

- **职责**：记录健康行为并生成报告。
- **核心设计**：
  - 记录类型当前包含：喝水（饮水量）、起身运动（分钟）、午睡（分钟），设计上类型可扩展（`type` 字段 + 对应 `unit`）。
  - **每日目标**：用户可自定义每种类型的每日目标值。
  - **每日报告**：按日期聚合当日各类型累计值与目标完成度。
  - **后续 AI 健康分析**：基于打卡数据生成建议（本期仅预留数据接口与扩展位）。

### 3.6 操作日志模块（Log）

- **职责**：记录用户操作（含登录）并提供可视化报告。
- **核心设计**：
  - 通过中间件自动记录请求（用户、动作、方法、路径、IP、UA、时间）。
  - 登录日志作为操作日志的一个类别（`category = login`）单独成表或通过分类筛选。
  - **报告**：按天、按小时聚合操作次数，返回柱状图所需数据结构。

### 3.7 接口限流模块（Rate Limit）

- **职责**：同一接口，每个 IP 每秒仅允许访问 1 次。
- **实现**：基于内存令牌桶 / 滑动窗口（key = `IP + 路由`），超限返回 `429`。
- **注意**：可按接口粒度配置，登录、发送验证码等敏感接口可加严。

## 4. 数据库设计

- 数据库：**SQLite**，通过 GORM 操作，统一软删除（`deleted_at`）。
- 命名约定：表名小写下划线（如 `task_groups`）；时间字段 `created_at` / `updated_at` / `deleted_at` 由 GORM 自动维护。

### 4.1 ER 关系概览

```
users  1 ──── ∞  phrases
users  1 ──── ∞  task_groups ──── self(nesting via parent_id)
task_groups 1 ── ∞  tasks
users  1 ──── ∞  check_in_records
users  1 ──── ∞  check_in_goals
users  1 ──── ∞  operation_logs
```

## 5. 接口设计

### 5.1 通用约定

- **Base URL**：`/api/v1`
- **请求格式**：`application/json`（部分查询参数走 query）。
- **统一响应格式**：

```json
{
  "code": 0,
  "message": "ok",
  "data": { }
}
```

- **鉴权**：除注册/登录/发送验证码/忘记密码等公开接口外，其余请求需在 Header 携带 `Authorization: Bearer <access_token>`。
- **限流**：所有接口每 IP 每秒 1 次，超限 `HTTP 429`，`code = 4290`。
- **错误码**：

| code | 含义 |
| --- | --- |
| 0 | 成功 |
| 4000 | 参数错误 |
| 4010 | 未认证 / Token 失效 |
| 4011 | Refresh Token 失效，需重新登录 |
| 4030 | 无权限（非本人资源） |
| 4040 | 资源不存在 |
| 4090 | 资源冲突（如邮箱已注册） |
| 4290 | 请求过于频繁（限流） |
| 5000 | 服务器内部错误 |

### 5.2 认证接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/auth/send-code` | 发送邮箱验证码（body: email, type） |
| POST | `/auth/register` | 注册（email, password, code） |

| POST | `/auth/refresh` | 刷新 access_token（refresh_token） |

| POST | `/auth/forgot-password` | 忘记密码：校验验证码并设置新密码 |
| PUT | `/auth/password` | 个人设置：改密（需旧密码或验证码） |
| GET | `/auth/profile` | 获取当前用户信息 |

## 6. 安全设计

| 风险 | 措施 |
| --- | --- |
| 密码泄露 | bcrypt 哈希存储，禁止明文 |
| Token 泄露 | Access Token 短有效期；Refresh Token 轮换 + 服务端可吊销 |
| 越权访问 | Repository 层强制 `user_id` 过滤；资源归属校验返回 4030/4040 |
| 暴力破解 / 爬虫 | 接口限流（每 IP 每秒 1 次）；敏感接口加严 |
| 验证码滥用 | 验证码有效期 + 一次性使用 + 发送频率限制 |
| SQL 注入 | 全部走 GORM 参数化查询，禁止拼接 SQL |
| XSS | 模板默认转义；用户内容输出时转义 |

## 7. 前端 / 模板说明

- 使用 `html/template`（标准）渲染基础页面骨架，复杂区块用 Jet 模板增强。
- 页面清单：登录 / 注册 / 忘记密码、个人设置、常用语管理、任务列表（左右两栏）、打卡与报告、操作日志与图表。
- 交互：任务编辑弹框（含常用语插入光标位置）、列表排序、复制到剪贴板均由前端 JS 实现。
- 图表：操作报告柱状图可用轻量库（如 Chart.js）渲染后端聚合数据。
