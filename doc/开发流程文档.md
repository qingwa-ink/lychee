# 荔枝小秘书（Lychee Assistant）— 开发流程文档

> 本文档描述从零搭建到上线的开发流程，配合 [项目设计文档](./项目设计文档.md) 使用，按里程碑推进即可。

---

## 1. 环境准备

| 项 | 要求 |
| --- | --- |
| Go | >= 1.21 |
| 数据库 | SQLite（无需额外服务，文件型） |
| 包管理 | Go Modules |
| 热重载（可选） | [air](https://github.com/cosmtrek/air) |
| 邮件 | SMTP 账号（用于发送验证码） |

**核心依赖**：

```bash
go get -u github.com/gin-gonic/gin
go get -u gorm.io/gorm
go get -u gorm.io/driver/sqlite
go get -u github.com/golang-jwt/jwt/v5
go get -u github.com/spf13/viper
go get -u golang.org/x/crypto/bcrypt
# 模板（按需）
go get -u github.com/cloudwego/jet
```

---

## 2. 项目初始化

```bash
go mod init github.com/qingwa-ink/lychee
```

按 [项目设计文档 §2.2](./项目设计文档.md#22-目录结构) 创建目录骨架，并新建 `config.yaml`：

```yaml
app:
  port: 8080
  env: dev            # dev / prod
jwt:
  secret: "change-me"
  access_ttl: "15m"
  refresh_ttl: "168h" # 7 天
rate_limit:
  per_second: 1       # 每 IP 每秒
mail:
  host: ""
  port: 465
  user: ""
  pass: ""
i18n:
  default: "zh"
db:
  path: "./data/lychee.db"
```

## 3. 开发里程碑

> 每个里程碑都是可独立验证的增量，建议按顺序推进。

### M1 — 项目骨架与基础设施

- 初始化目录结构、`main.go`、配置加载、日志。
- 连接 SQLite + GORM，注册 AutoMigrate（见设计文档 §4）。
- 统一响应封装（`code/message/data`）与错误码。
- 健康检查接口 `GET /health`。

**验收**：服务启动、迁移建表成功、`/health` 返回正常。

### M2 — 认证模块（核心）

- 发送邮箱验证码（含有效期、一次性、频率限制）。
- 注册、登录（bcrypt + 签发 Access/Refresh Token）。
- 刷新接口（Refresh Token 轮换）、登出。
- JWT 中间件 + Repository 层 `user_id` 隔离。

**验收**：完成注册→登录→刷新→登出全链路；越权访问被拦截。

### M3 — 忘记密码与个人设置

- 忘记密码（验证码 + 重置）。
- 个人设置：修改密码。

**验收**：可用验证码重置密码并重新登录。

### M4 — 多语言

- 文案文件 `zh.yaml` / `en.yaml`。
- i18n 中间件（显式切换 > `Accept-Language` > 默认）。
- 模板注入 `t(key)`，前端语言切换写 Cookie。

**验收**：中英文切换生效并持久化。

### M5 — 常用语管理

- CRUD 接口 + 列表页。
- 编辑任务时可选常用语插入光标位置（前端联调留口）。

**验收**：增删改查正常，数据按用户隔离。

### M6 — 任务管理（重点）

- 分组树（自引用嵌套）接口 + 左侧分组面板。
- 任务 CRUD + 排序（创建/修改/截止/优先级）。
- 编辑弹框（内容、优先级、状态）+ 常用语插入 + 复制到剪贴板。
- 删除分组级联软删除。

**验收**：左右两栏联动、嵌套分组、排序、编辑弹框、复制均可用。

### M7 — 打卡与健康

- 打卡记录新增 / 查询（按日期、类型）。
- 每日目标设置、每日报告（累计 + 完成度）。
- 预留 AI 健康分析扩展位。

**验收**：记录打卡、查看每日报告与目标完成度。

### M8 — 操作日志与报告

- 操作日志中间件（自动落库，含登录）。
- 操作历史接口（分页、时间筛选）。
- 报告接口（每日/每小时聚合）+ 前端柱状图。

**验收**：操作可追溯，图表正确展示操作频次。

### M9 — 限流、收尾与部署

- 接口限流中间件（每 IP 每秒 1 次，敏感接口加严）。
- 全局错误处理、日志、配置项复核。
- 生产构建、部署、SQLite 持久化目录确认。

**验收**：压测限流生效；生产环境稳定运行。

## 4. 编码规范

- **分层**：Controller 只做参数校验与响应组装，业务放 Service，数据访问放 Repository。
- **错误处理**：Service 返回带错误码的业务错误；Controller 统一转换为响应结构。
- **数据隔离**：所有 Repository 方法签名包含 `userID`，查询强制带 `WHERE user_id = ?`。
- **命名**：包名小写单数；模型用大驼峰，字段 GORM 自动蛇形映射。
- **配置**：敏感信息（密钥、SMTP）走环境变量覆盖 `config.yaml`。
- **Context**：请求作用域值（`userID`、`locale`）通过 `gin.Context` / `context.Context` 传递。

## 5. 测试策略

| 层级 | 范围 | 工具 |
| --- | --- | --- |
| 单元测试 | Service / Repository 业务逻辑与隔离 | `testing` + SQLite 内存库 |
| 接口测试 | 关键 API（注册/登录/刷新/任务/打卡） | `httptest` |
| 重点用例 | 越权访问、Token 过期、限流、验证码失效 |  |

目标：核心认证与任务模块接口测试覆盖。

---

## 6. 配置与环境变量

- `config.yaml` 提供默认值；部署时用环境变量覆盖：
  - `APP_PORT`、`JWT_SECRET`、`MAIL_HOST/USER/PASS`、`DB_PATH`、`I18N_DEFAULT`。
- 生产环境务必替换 `JWT_SECRET`。

---

## 7. 构建与部署

```bash
# 构建
CGO_ENABLED=1 go build -o bin/lychee ./cmd/server

# 运行
./bin/lychee
```

- SQLite 数据文件默认 `./data/lychee.db`，部署时挂载持久化目录。
- 可前置 Nginx 做反向代理与静态资源托管。
- 建议用 systemd / supervisor 守护进程。
